# Release v0.2.5 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish consistent local release metadata for version `0.2.5` and mark its commit with annotated tag `v0.2.5`.

**Architecture:** The root `package.json` remains the runtime-readable version source because the Dockerfile copies it into `/app`. A matching annotated Git tag identifies the exact source commit; no runtime code or dependencies change.

**Tech Stack:** Node.js, pnpm, Git, JSON package metadata

## Global Constraints

- Use package version `0.2.5` and Git tag `v0.2.5`.
- Preserve all existing historical tags, including `v0.2.4.1`.
- Do not change dependencies or `pnpm-lock.yaml`.
- Create the tag locally; do not push commits or tags without explicit authorization.
